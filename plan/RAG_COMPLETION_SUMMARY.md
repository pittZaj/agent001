### 问题 4：知识库暴露形式

**明确回答**：知识库是作为一个 **Skill（技能）** 暴露的，具体说明：

#### 暴露形式
- **类型**：`SkillType.TOOL`（本地计算类）
- **注册名**：`kb_regulation`
- **注册位置**：`skills/kb/skill.py` → `skills/init.py` 统一注册到 Skill Registry

#### 工作流程（自动意图识别）
```
用户问题："未戴安全帽违反哪些规定？"
   ↓
Planner（LLM）自动判断 → 需要检索知识库
   ↓
生成包含 kb_regulation 的 plan
   ↓
Executor 调用 kb_regulation Skill → 检索 Qdrant
   ↓
Formatter 综合检索结果 → 生成自然语言回复
```

#### 主智能体已具备知识库能力
- 当前主智能体的 Skill Registry 已注册 `kb_regulation`
- **不需要单独构造知识库问答智能体**
- 只需测试用例即可展示联动功能

#### 其他智能体如何使用知识库
- 通过 **Agent Spec 配置**（Tab 1 新建任务时的"可用工具"里是否包含 `kb_regulation`）
- 可以手动决定某个 Agent 是否能用知识库、在哪步用
- 这是 Skill Registry 架构的设计意图

**技术优势**：
1. **自动意图识别**：Planner（LLM）会自动判断用户问题是否需要检索知识库，无需手动指定
2. **统一管理**：所有 Skill（数据访问、本地计算、知识库）都在 Skill Registry 中统一管理
3. **灵活配置**：新增 Agent 时可自由选择是否启用知识库功能

## 三、技术实现总结

### 3.1 知识库架构

```
LangGraph 主图 (Planner → Executor → Formatter)
  ↓
Skill Registry (统一工具注册表)
  ↓
kb_regulation Skill (SkillType.TOOL)
  ↓
KnowledgeBaseService (skills/kb/service.py)
  ↓
Qdrant (localhost:6333, collection=safety_regulations)
```

### 3.2 核心技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 向量数据库 | Qdrant | 轻量化，< 2GB，支持百万级向量 |
| 词嵌入模型 | BGE-M3 | 本地加载，dim=1024，sentence-transformers |
| 重排序模型 | BGE-reranker-v2-m3 | FlagEmbedding，fp16 加速，GPU |
| 文档解析 | Unstructured | 支持 PDF/Word/TXT/Markdown |
| 分块策略 | RecursiveCharacterTextSplitter | 固定大小/按段落/按标题 |
| 检索模式 | 混合检索（hybrid） | 语义 + 关键词，效果最好 |

### 3.3 环境约束（关键）

- **torch**：2.8.0+cu128（钉版本，否则 cuda.is_available()==False）
- **transformers**：4.57.6（钉版本，5.x 会报 `prepare_for_model` 错误）
- **设备号**：agent 环境中 5880 显卡的设备号是 `cuda:0`
- **模型路径**：BGE-M3=`/mnt/data3/clip/LangGraph/VLLM/BGE-M3`，reranker=`/mnt/data3/clip/LangGraph/VLLM/bge-reranker-v2-m3`
- **离线环境**：`HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1`（服务器无法访问 huggingface.co）

### 3.4 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| chunk_size | 300 | 固定大小分块的字符数 |
| chunk_overlap | 50 | 重叠字符数 |
| retrieval_mode | hybrid | 混合检索（语义+关键词） |
| default_top_k | 5 | 召回数量 |
| recall_multiplier | 4 | 召回 top_k * 4 后重排 |
| score_threshold | 0.5 | 重排序最低分（0.5-0.7） |
