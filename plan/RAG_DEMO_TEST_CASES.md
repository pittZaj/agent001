# RAG 知识库联动 - 演示测试用例

**目的**：向领导展示智能体平台已完整支持知识库功能，能够检索规章制度并与告警业务联动。

**测试环境**：
- 知识库：已上传 2 个文档（安全生产管理规定 + 员工手册），共 138 个向量
- 主智能体：已注册 `kb_regulation` Skill，可自动识别是否需要检索知识库
- Web 控制台：Gradio Tab6（主智能体对话）+ Tab7（知识库管理）

---

## 一、知识库管理演示（Tab7）

### 测试用例 1.1：文档上传

**操作步骤**：
1. 打开 Tab7「知识库管理」
2. 点击「选择文件」，上传 `plan/安全生产管理规定.md`
3. 填写标题："安全生产管理规定 V2.0"
4. 选择分类："安全规定"
5. 选择分块策略："fixed_size"（固定 300 字）
6. 点击「上传」

**预期结果**：
- ✅ 上传成功
- 显示文档 ID 和分块数（约 14 个分块）
- 分块策略：fixed_size

**业务价值**：支持 PDF/Word/Markdown/TXT 多种格式，灵活的分块策略（固定大小/按段落/按标题）。

### 测试用例 1.2：检索测试（参数调优）

**操作步骤**：
1. 在「检索测试」区域输入查询："未戴安全帽违反哪些规定？会被怎么处罚？"
2. 设置召回数量：5
3. 选择检索模式："hybrid"（语义+关键词，效果最好）
4. 设置相似度阈值：0.5
5. 点击「检索」

**预期结果**：
- 返回 2-3 条相关结果
- 第 1 条（相关度 0.96+）：完整引用第一条安全帽佩戴规定，包含国家标准 GB 2811-2019 和处罚标准（警告→罚款 50 元→停工 3 天→解聘）
- 第 2 条（相关度 0.93+）：补充说明适用告警类型 `no_helmet`
- 每条结果显示：来源、分类、文档 ID、相关度分数、内容片段

**业务价值**：
- 混合检索（语义+关键词）效果优于纯语义检索，能精准定位条文
- 相似度阈值可调（0.5-0.7），过滤不相关内容
- 检索结果包含完整出处，可追溯到原文

## 二、主智能体联动演示（Tab6）

### 测试用例 2.1：安全规章查询（告警类型相关）

**操作步骤**：
1. 打开 Tab6「Agent 对话测试」
2. 下拉框选择「主智能体(增强)」
3. 输入："未戴安全帽违反哪些规定？会被怎么处罚？"
4. 点击发送

**预期效果**（20-30s）：
- Planner 自动识别需要检索知识库，生成包含 `kb_regulation` 的 plan
- Executor 调用 `kb_regulation` Skill，检索到相关条文
- Formatter 综合检索结果，生成自然语言回复：
  - 引用《安全生产管理规定》第一条
  - 明确列出国家标准 GB 2811-2019
  - 详细说明 4 级处罚标准（警告 → 罚款 50 元 → 停工 3 天 → 累计 5 次解聘）
  - 注明适用告警类型：`no_helmet`

**调试面板验证**：
- Plan 中包含 `kb_regulation` 步骤
- Tool Results 显示检索到的条文片段
- 无幻觉（所有信息均来自检索结果，不是 LLM 编造）

**业务价值**：
- **告警 + 规章联动**：VLM 识别"未戴安全帽"告警 → 自动查询相关规定 → 完整闭环
- **可追溯性**：回复引用具体条文出处，满足合规审计要求
- **无幻觉**：基于真实文档检索，不会编造不存在的规定

## 三、联动场景演示（告警 + 规章）

### 测试用例 3.1：复判 + 规章联动

**操作步骤**：
1. 准备一个 `pending` 状态的 `no_helmet` 告警（从测试数据中选取 UUID）
2. 输入："复判告警 `<UUID>`，如果确实违规，告诉我违反了哪些规定"
3. 点击发送

**预期效果**（30-40s，含 VLM 推理）：
- 步骤 1：VLM 复判图像，输出 `verdict=confirmed, confidence=0.95`
- 步骤 2：Planner 识别需要查询规章，调用 `kb_regulation`
- 最终回复：
  - 复判结论：确认违规，置信度 95%
  - 违反规定：《安全生产管理规定》第一条
  - 处罚标准：首次警告 → 再犯罚款 50 元 → 三犯停工 3 天 → 累计 5 次解聘
  - 国家标准：GB 2811-2019
  - 告警类型：`no_helmet`

**业务价值**：
- **端到端闭环**：图像识别 → 复判 → 查规章 → 处罚建议
- **可解释性**：每个环节都有依据（VLM 推理理由 + 规章条文引用）
- **自动化**：人工只需确认，无需手动查规章

## 六、常见问题预案

### Q1: 知识库检索速度如何？
**A**: 单次检索延迟 < 500ms（含词嵌入 + 重排序），满足实时查询要求。批量检索（如测试用例 3.2）耗时 40-50s，因为需要多次调用。

### Q2: 知识库容量上限？
**A**: Qdrant 单机可支持百万级向量。当前 138 个向量（2 个文档），容量远未饱和。实际生产可上传 50-100 个规章文档，预计 5000-10000 个向量。

### Q3: 如何保证检索结果准确性？
**A**: 
1. **混合检索**（语义+关键词）效果优于纯语义，减少漏召回
2. **相似度阈值**过滤低分结果（默认 0.5，可调至 0.6-0.7）
3. **重排序**模型（BGE-reranker-v2-m3）二次精排，提升 Top-K 准确率
4. **人工审核**：知识库内容可查看/编辑/删除，确保源头正确

### Q4: 是否需要单独构造知识库问答智能体？
**A**: **不需要**。知识库是作为 **Skill** 暴露的，主智能体已注册 `kb_regulation`。Planner 会自动判断用户问题是否需要检索知识库（意图识别），无需单独构造智能体。

### Q5: 其他智能体如何使用知识库？
**A**: 通过 **Agent Spec 配置**。在 Tab1 新建任务时，"可用工具"里勾选 `kb_regulation`，该智能体就能使用知识库。这是 Skill Registry 架构的设计意图。

### Q6: Word 文档上传报错怎么办？
**A**: 已修复。执行 `pip install "unstructured[doc]"` 安装依赖即可。PDF、Markdown、TXT 无此问题。

## 七、后续优化方向

1. **文档版本管理**：同一规章的多个版本（V1.0、V2.0），支持查询"最新版"或"历史版本"
2. **权限控制**：不同角色看到不同规章（如普通员工看不到高管薪酬规定）
3. **多模态检索**：上传带图表的规章文档，支持图文混合检索
4. **自动更新**：规章修订后自动重新上传，保持知识库最新
5. **问答日志分析**：统计高频查询，优化文档分块与检索策略
